# Skill: 平衡控制通用原理

## 适用范围

所有需要"主动平衡"的机器人——双足、自平衡轮式、单轮、动态四足。
不适用于:重心稳定的静态机器人(典型 6 轮 AGV、固定臂)。

## 平衡控制系统的三大支柱

任何平衡控制系统由三个子系统协同:

```
                ┌─────────────────────┐
                │    控制目标          │
                │   (姿态 / 运动)     │
                └──────────┬──────────┘
                           │
      ┌────────────────────┼────────────────────┐
      ▼                    ▼                    ▼
┌──────────┐        ┌──────────┐         ┌──────────┐
│ 感知层    │  →     │ 估计层    │   →    │ 控制层    │
│ IMU      │        │ 状态估计  │         │ 关节伺服  │
│ 力传感器 │        │ 位姿融合  │         │ 步态生成  │
└──────────┘        └──────────┘         └──────────┘
```

任一支柱出问题,平衡都会受影响。**诊断不稳必须按这三层依次排查**。

## 三步诊断方法

### 第一步:感知层(IMU + 力)

诊断从最底层开始,任何上层判断必须基于可信数据:

- IMU 数据是否有效?
- 数据频率是否符合规格?
- 静止时的零偏是否在合理范围?
- 力传感器读数是否一致?

如果感知层不可信,**重新标定再继续**,跳过这一步会让所有后续推理错误。

### 第二步:估计层(状态融合)

感知数据可信后,看融合是否正确:

- 估计的位姿是否与肉眼观察一致?
- 速度估计是否漂移?
- 估计置信度是否健康?

如果估计层异常,**重启估计器或重新标定**。

### 第三步:控制层(关节响应)

估计可信后,看控制器是否能跟随:

- 各关节跟踪误差是否在合理范围?
- 关节响应延迟是否符合规格?
- 关节力矩是否饱和?

控制层问题通常是**机械问题**或**控制参数问题**。

## 常见的"伪平衡问题"

不是所有"看起来不稳"的现象都需要平衡修复,有几类要先排除:

| 现象 | 实际原因 | 处理 |
|---|---|---|
| 站立时姿态缓慢漂移 | IMU 零偏 | 标定 IMU |
| 行走偏向单侧 | 估计器零偏或左右传感器不平衡 | 标定 + 检查 |
| 摔倒后无法站起 | 不是平衡问题,是恢复任务 | 执行起身流程 |
| 在湿滑地面打滑 | 接触模型(摩擦)问题 | 调步态参数,不是平衡问题 |

诊断"步态不稳"时**先排除上述四类**,免得把感知/标定/恢复问题当成平衡问题处理。

## 地面与环境因素的独立性

地面问题会让平衡控制器**做出正确的反应但仍然失败**——
此时控制器不是坏的,环境超出设计范围。

地面相关因素:
- 摩擦系数(瓷砖、木地板、水泥、地毯各不同)
- 平整度(不平整 > 步高时无法稳定)
- 软硬(地毯太软会吸收冲击,影响触地反馈)
- 倾斜
- 清洁度(灰、油、水)

诊断"在某种地面不稳"时,不要去改控制器,**先调地面参数**(摩擦/步态),
更换适合的鞋底材质,或者标记该地面为"不可使用"。

那些放具体机器人的 instance 层。
