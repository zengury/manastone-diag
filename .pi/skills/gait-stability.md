# Skill: 双足步态稳定性

## 适用范围

双足人形机器人。

## 双足平衡的特殊难度

双足是动态平衡机器人中最难的:

- **支撑多边形小**(只有两脚之间的区域,占地小)
- **冲击负载大**(每一步落地都是脉冲)
- **质心位置高**(头部/躯干位置高,杠杆作用大)
- **耦合度高**(任一腿的关节问题影响全局平衡)

实际意义:**任何单腿单关节问题,都会显现为整体不稳**。诊断时不要只看不稳的方向。

## 双足步态参数(关键)

理解这几个参数对诊断很重要:

| 参数 | 含义 | 调小的影响 | 调大的影响 |
|---|---|---|---|
| step_height | 抬腿高度 | 摩擦小、容易绊倒 | 能耗大、姿态不稳 |
| step_length | 步长 | 慢、稳 | 快、风险大 |
| stance_width | 双脚间距 | 转向灵活 | 横向稳定 |
| swing_time | 摆动腿时长 | 高频步态 | 慢节奏 |
| double_support | 双脚同时着地占比 | 灵活、像"奔跑" | 稳定、像"踱步" |

诊断时:
- 不稳定 → 增大 stance_width + 减小 step_height + 增大 double_support
- 摔倒 → 同上,极端时降到"踱步"
- 太慢 → 反向调

## 双足摔倒模式分类

不同摔倒原因对应不同处置:

| 摔倒模式 | 特征 | 应对 |
|---|---|---|
| 前向摔倒(plant face) | 步长过大 / 减速失败 | 减步长、增 double_support |
| 后向摔倒 | 重心过后 / 突然刹车 | 调整重心位置 |
| 侧向摔倒 | stance_width 不足 / 单腿支撑期过长 | 增 stance_width |
| 原地崩溃 | 平衡控制器失效 | 检查 IMU + 估计器 |
| 突发摔倒(无原因) | 关节断开 / 力矩中断 | 检查关节状态 |

## 双足在 universal 三步基础上的扩展

### 第一步扩展:除 IMU 外,看脚力分布

双足的脚力传感器(双脚 4 个力传感器)是平衡控制的关键输入:

- 双脚总力 ≈ 整机重量
- 左右脚力比例反映重心位置
- 单脚内部前后/左右力比例反映踝关节姿态

如果脚力数据异常,IMU 再准也无法平衡。

### 第二步扩展:估计器与脚力的耦合

双足的状态估计依赖**接触感知**——估计器需要知道哪只脚正在接地。

如果接触感知错误(误判触地或误判悬空):
- 摆动腿被当成支撑腿 → 摔
- 支撑腿被当成摆动腿 → 撑不住

诊断估计器漂移时,**额外检查接触感知**。

### 第三步扩展:对称关节响应分析

参考 humanoid/joint-thermal-management 的"左右对称是金标准"——
双足的关节响应也用相同方法诊断:对比左右对称关节的跟踪误差,显著差异表示
某侧关节有机械/控制问题。

## 双足特定的"伪平衡问题"

补充 universal 列表:

| 现象 | 双足实际原因 |
|---|---|
| 站立时左右晃动 | 通常是 stance_width 不足或重心估计偏差 |
| 行走时上下颠簸 | 步态参数中 step_height 不当或膝关节阻尼不足 |
| 转向不顺 | hip yaw 关节响应慢或地面摩擦不足 |
| 楼梯失稳 | 步高未匹配实际台阶高度 |
