# 模式切换异常与摔倒

## 触发条件

- 机器人从站立/行走模式突然摔倒
- 日志中检测到 `Detect Falling Action`
- 短时间内（< 2秒）有多次模式切换请求

## 诊断步骤

1. **确认摔倒时间和原因**
   - 搜索 `Detect Falling Action` —— 确认摔倒精确时刻
   - 查看 torso/chest tilt angle 变化速率

2. **回溯摔倒前 5 秒的模式切换**
   - 搜索 `设置ACTION切换` / `SetMcAction` / `目标Action`
   - 特别注意：`JOINT_DEFAULT`（位控模式）在站立状态下是危险操作
   - 特别关注：短时间内大量并发请求（洪水式）→ 遥控器误操作特征

3. **对照 ontology 模式迁移规则**
   - 读取 `knowledge/actions.yaml` 和 `knowledge/hardware.yaml`
   - 检查摔倒前的模式切换是否符合 ontology 的允许迁移路径
   - X2 模式迁移约束：部分模式不允许从站立状态直接切换

4. **区分人因与系统故障**
   - 多个线程同时请求同一模式切换 → 遥控器按键卡住/误触
   - 模式切换间隔随机 → 人为操作
   - 固定间隔重复 → 系统循环错误

## 处理方案

- **确认遥控器误操作**: 检查操作员培训，考虑增加模式切换确认步骤
- **系统防护**: 在位控模式下增加站立状态检测，自动拒绝危险切换
- **记录**: 沉淀为诊断案例，更新 capability_boundary
