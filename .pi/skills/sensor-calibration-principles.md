# Skill: 传感器标定通用原理

## 适用范围

所有依赖传感器的机器人。

## 标定的本质:消除"已知误差"

传感器标定不是修传感器,是**让控制系统知道传感器有多偏**,然后软件层补偿掉。

不同传感器的偏差类型:

| 传感器 | 主要偏差类型 | 标定动作 |
|---|---|---|
| IMU | 零偏(零速时仍有读数) | 静置标定零偏 |
| 加速度计 | 重力轴对齐偏差 | 多姿态标定 |
| 陀螺仪 | 零偏 + 温度漂移 | 静置 + 预热 |
| 力传感器 | 零点偏移、量程刻度 | 已知重量标定 |
| 编码器 | 零位偏差 | 机械零位对齐 |
| 磁强计 | 软铁/硬铁误差 | 8 字标定 |

**任何标定都基于"已知输入"**。没有已知输入的标定不存在。

## 标定的"前置条件"是关键

绝大多数标定失败,根因是**前置条件不满足**:

| 标定类型 | 关键前置条件 |
|---|---|
| IMU 零偏 | 绝对静止 + 绝对水平 + 温度稳定 |
| 力传感器 | 已知重量 + 受力方向已知 |
| 编码器零位 | 机械结构在指定姿态 |
| 磁强计 | 远离铁磁干扰 |

诊断"标定失败"时,**永远先怀疑前置条件**,再怀疑算法或硬件。

## 标定结果的验证

标定完成不代表标定成功,必须验证:

| 标定 | 验证方法 |
|---|---|
| IMU 零偏 | 静置时输出应接近零(零偏 < 阈值) |
| 力传感器 | 不施加力时读数为零;施加已知力时读数符合 |
| 编码器零位 | 关节位于零位时反馈为零 |
| 磁强计 | 旋转一周,模长应保持恒定 |

**没验证的标定不能上线**。

## 标定周期

不同传感器的"老化"速度差很多:

| 传感器 | 周期建议 |
|---|---|
| IMU | 每月一次,运输后必标 |
| 力传感器 | 每月一次 |
| 编码器 | 一旦机械变化必标(更换、碰撞) |
| 磁强计 | 环境变化时(进新场地) |
| 摄像头内参 | 半年到一年 |
| 摄像头外参(多相机) | 同上,机械改动后必标 |

## 标定数据管理

标定参数是**机器人个体身份**的一部分:

- 不同机器人的标定参数不能互换
- 标定参数必须备份(机器损毁能恢复个体身份)
- 标定参数版本化管理(回退能力)
